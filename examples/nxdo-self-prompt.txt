`nxdo` is a Python package that inspects the current project state, reads recent git history, adds a user question for an LLM, and returns a concrete plan for the next 10 engineering tasks.

**Documentation:** [docs/](docs/) В· **Examples:** [examples/](examples/) В· **Step-by-step guide:** [docs/how-it-works.md](docs/how-it-works.md)

## How it works (step by step)

1. **Analyze the repo** вЂ” README, manifests, tree, stack (`project_analyzer`)
2. **Read git context** вЂ” commits, changed files, TODO/FIXME markers (`git_reader`)
3. **Optional metrics** вЂ” complexity, coupling, hotspots (`metrics`)
4. **Build prompt** вЂ” snapshot + git + your `--extra-context` (`llm_client`)
5. **Call LLM** вЂ” OpenAI-compatible API, default OpenRouter (`providers`)
6. **Validate plan** вЂ” 10 tasks as Pydantic `TaskPlan` (`models`)
7. **Output** вЂ” terminal, JSON, TODO.md, or `.planfile/` (`tickets`, `auto`)

```bash
pip install nxdo
export OPENROUTER_API_KEY="your-key"

# 1вЂ“2: inspect context (no API cost)
nxdo print-context .

# 3: metrics only (no API cost)
nxdo metrics .

# 4вЂ“6: generate plan
nxdo plan . -e "What should we build next?"

# 7: export or sync tickets
nxdo plan . --json > plan.json
nxdo tickets . --sync-planfile
```

Real outputs from running nxdo **on this repo**: [examples/nxdo-self-plan.json](examples/nxdo-self-plan.json) В· [examples/nxdo-self-context.txt](examples/nxdo-self-context.txt) В· [examples/nxdo-self-metrics.txt](examples/nxdo-self-metrics.txt)

See the full walkthrough in [docs/how-it-works.md](docs/how-it-works.md).

## What is included

- **Project snapshot analysis** вЂ” README, manifests (pyproject.toml, package.json, Cargo.toml, вЂ¦), directory tree, stack detection
- **Git context** вЂ” recent commits, most-changed files, TODO/FIXME markers
- **Advanced code metrics** вЂ” cyclomatic complexity, coupling analysis, bug hotspots, bus factor detection
- **Koru-aware planning** вЂ” Deep integration with koru framework for intelligent task generation
- **Pydantic models** вЂ” validated data models for tasks and plans (`Task`, `TaskPlan`)
- **Provider abstraction** вЂ” pluggable LLM backends; ships with an OpenAI-compatible provider (works with OpenRouter and any OpenAI-style API)
- **Planner orchestrator** вЂ” `generate_next_tasks()` composes analysis + prompt + LLM
