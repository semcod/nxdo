# nxdo examples

Real outputs generated by running nxdo **on the nxdo repository itself**.

| File | Command | Description |
|------|---------|-------------|
| [nxdo-self-context.txt](./nxdo-self-context.txt) | `nxdo print-context . --max-commits 5 --raw` | Project snapshot + git context (no LLM) |
| [nxdo-self-metrics.txt](./nxdo-self-metrics.txt) | `nxdo metrics . --top 5` | Complexity, coupling, hotspots (no LLM) |
| [nxdo-self-prompt.txt](./nxdo-self-prompt.txt) | `nxdo print-prompt . -e "…" --max-commits 5` | Full prompt sent to the LLM (no LLM call) |
| [nxdo-self-plan.json](./nxdo-self-plan.json) | `nxdo plan . -e "…" --max-commits 10 --json` | Validated 10-task `TaskPlan` JSON |
| [nxdo-self-plan.txt](./nxdo-self-plan.txt) | `nxdo plan . -e "…" --max-commits 10` | Human-readable Rich plan output |

## Verify examples

```bash
./examples/check-examples.sh
```

This checks that all files exist, contain expected nxdo markers, JSON has 10 tasks, and `nxdo validate` passes.

## Regenerate

```bash
cd /path/to/nxdo
export OPENROUTER_API_KEY=...

EXTRA='What are the next sensible engineering steps for nxdo?'

nxdo print-context . --max-commits 5 --raw > examples/nxdo-self-context.txt
nxdo metrics . --top 5 > examples/nxdo-self-metrics.txt
nxdo print-prompt . -e "$EXTRA" --max-commits 5 > examples/nxdo-self-prompt.txt
nxdo plan . -e "$EXTRA" --max-commits 10 --json > examples/nxdo-self-plan.json
nxdo plan . -e "$EXTRA" --max-commits 10 > examples/nxdo-self-plan.txt

./examples/check-examples.sh
```

## Validate the sample plan

```bash
nxdo validate examples/nxdo-self-plan.json
```
